# 할인율 진단 텔레봇 (Discount Diagnostics Bot)

목표가 컨센서스 대비 현재 주가의 할인율(괴리율)을 계산하고, 그 할인이 어떤 요인(매크로/수급/업황·모멘텀/멀티플/거버넌스)에서 오는지 기여도를 배분해 텔레그램으로 회신하는 봇. 매크로 대시보드·리서치 에이전트와 동일한 스택 (Python + GitHub Actions + Telegram Bot API, 전부 무료).

## 사용법

```
/discount 005930   # 삼성전자 진단
/d 000660          # 단축 명령
/help
```

## 방법론

- **총 할인율** = (목표가 컨센서스 − 현재가) / 목표가
- **요인 압력 점수**: 9개 요인을 각각 0~100으로 정규화 (높을수록 할인 압력)
- **기여도 배분**: 총 괴리율 × (가중치 × 압력) / Σ(가중치 × 압력)
- **할인 성격 분해**: 시점 할인(매크로+수급+모멘텀, 사이클 해소형) vs 구조 할인(멀티플+거버넌스, 이벤트 필요형)

> ⚠️ 기여도는 인과 분해가 아닌 휴리스틱 배분. 회귀 기반 정밀 귀속은 진단 이력 축적 후 v2에서 가능.

## 배포 (GitHub Actions)

1. 새 private repo 생성 후 이 파일들 push
2. Settings → Secrets and variables → Actions에 등록:
   - `TELEGRAM_BOT_TOKEN` (기존 봇 재사용 가능, 단 리서치 에이전트와 같은 봇이면 getUpdates offset 충돌 → **별도 봇 신규 발급 권장**, @BotFather 무료)
   - `TELEGRAM_CHAT_ID` (실패 알림 수신용)
3. Actions 탭에서 workflow 활성화 — 15분 크론 폴링 (workflow_dispatch로 즉시 실행 가능)

## 주의사항 / 알려진 한계

- **동일 봇 토큰 공유 금지**: 리서치 에이전트가 이미 getUpdates를 소비 중이면 이 봇이 메시지를 못 받음. 봇 토큰을 분리할 것.
- **네이버 API 스키마 변동 가능**: `collectors.py`의 `integration`/`frgn` 파싱은 스키마 변경 시 부분 실패할 수 있음 (실패 항목은 ⚪ 표시, 배분에서 자동 제외). 최초 배포 시 삼성전자(005930)로 실행해 각 항목이 채워지는지 검증 권장.
- **공매도 잔고**: KRX API 인증 이슈로 v1 제외. v2 후보.
- **KOSDAQ 종목**: yfinance에서 `.KS` 실패 시 `.KQ` 자동 재시도.
- **응답 지연**: Actions 크론 특성상 최대 15~20분. 즉시 필요하면 Actions 탭에서 수동 실행.
